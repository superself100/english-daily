02

핵심 용어 사전

처음 보는 용어들이지만 비유로 이해하면 쉬워요! 한 번만 읽으면 전체 흐름이 보입니다.
📦
JSON
택배 상자예요. Claude가 만든 기사·단어·퀴즈를 정해진 형식으로 담아서 GitHub으로 전달합니다.
{"ARTICLE_TITLE": "한타바이러스 발생", "W1EN": "outbreak", "W1KR": "발생"}
🪪
SHA
파일의 주민등록번호예요. GitHub가 "이 파일 맞냐?" 확인할 때 쓰는 고유 번호입니다. 같은 파일을 수정할 때만 필요해요.
SHA = "abc123def456..." (파일마다 고유한 값)
📡
RSS
뉴스 자동 구독 피드예요. BBC처럼 언론사가 최신 뉴스를 자동으로 제공하는 방식입니다. Make가 매일 이걸 가져와요.
feeds.bbci.co.uk/korean/rss.xml → 오늘 뉴스 20개 자동 제공
🚪
API
서비스끼리 대화하는 창구예요. Claude API = Claude와 대화할 수 있는 문. Make가 이 문을 두드려서 기사를 요청합니다.
Make → (API) → Claude: "기사 만들어줘!" → Claude가 JSON으로 응답
🧩
모듈
Make 비서의 업무 단계예요. 레고 블록처럼 연결해서 자동화 흐름을 만듭니다. 1번→8번→2번→6번 순서로 실행돼요.
1번(템플릿) → 8번(뉴스) → 2번(Claude) → 6번(저장)
🏭
GitHub Actions
파일이 저장되면 자동으로 실행되는 인쇄소예요. data.json + template.html을 합쳐서 index.html을 자동 생성합니다.
data.json 저장 감지 → template에 내용 채우기 → index.html 완성!
💾
localStorage
사용자 브라우저에 저장되는 개인 학습 기록 공간이에요. 같은 링크를 공유해도 각자의 기록이 따로 저장됩니다.
A의 브라우저: 학습일 15일 / B의 브라우저: 학습일 3일 (완전 분리)
🔄
GET / POST / PUT
GET=가져오기(뉴스 읽기), POST=처리요청(Claude한테 기사 만들어달라고), PUT=덮어쓰기(GitHub에 파일 저장)
식당 비유: GET=메뉴보기, POST=주문하기, PUT=주문변경

STEP 3

GitHub Actions 설정

GitHub → 저장소 → Add file → Create new file → 파일명: .github/workflows/build.yml
build.yml
name: Build Daily English Page
on:
  push:
    paths:
      - 'data-*.json'
jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - name: Build HTML files
        run: |
          python3 << 'EOF'
          import json, glob, re
          files = sorted(glob.glob('data-*.json'))
          fix_files = [f for f in files if 'fix' in f]
          latest = fix_files[-1] if fix_files else files[-1]
          date_str = re.search(r'data-(\d{4}-\d{2}-\d{2})', latest).group(1)
          with open(latest, 'r', encoding='utf-8') as f:
              data = json.load(f)

          # index.html 생성
          with open('template.html', 'r', encoding='utf-8') as f:
              html = f.read()
          for key, value in data.items():
              html = html.replace('{{' + key + '}}', str(value))
          with open('index.html', 'w', encoding='utf-8') as f:
              f.write(html)
          print(f"index.html 생성 완료")

          # article-날짜.html 생성 (과거 기사 보관용)
          import os
          if os.path.exists('article-template.html'):
              with open('article-template.html', 'r', encoding='utf-8') as f:
                  article_html = f.read()
              article_html = article_html.replace('{{ARTICLE_DATE}}', date_str)
              for key, value in data.items():
                  article_html = article_html.replace('{{' + key + '}}', str(value))
              article_filename = f'article-{date_str}.html'
              with open(article_filename, 'w', encoding='utf-8') as f:
                  f.write(article_html)
              print(f"{article_filename} 생성 완료")
          EOF
      - name: Commit files
        run: |
          git config user.name "github-actions"
          git config user.email "actions@github.com"
          git add index.html article-*.html
          git commit -m "Daily update" || echo "No changes"
          git push

자주 발생하는 오류 & 해결법

❌ JSON 오류
The provided JSON body is not valid JSON
원인: 프롬프트 안에 줄바꿈이나 특수문자가 JSON을 깨뜨림
해결: 프롬프트를 한 줄로 작성하거나 영어로만 작성
❌ SHA 오류
"sha" wasn't supplied
원인: 같은 날짜 파일이 이미 존재할 때 SHA 값 필요
해결: 파일명에 시간 추가 → data-YYYY-MM-DD-HHmm.json
❌ GitHub Actions 권한 오류
Process completed with exit code 128
원인: GitHub Actions가 파일을 push할 권한 없음
해결: Settings → Actions → General → Read and write permissions
❌ RSS 빈값 오류
RSS feed appears to be empty (1. 2. 3. 만 출력)
원인: Make의 RSS 모듈 변수명이 잘못 참조됨
해결: {{1.data.rss.channel[].item[1].title[]}} 형태로 수정 (1번 모듈 기준)
❌ 시나리오 자동 비활성화
Scenario was deactivated by Make
원인: 연속 오류 발생 시 Make가 자동으로 OFF
해결: 오류 수정 후 시나리오 토글 → ON으로 변경
